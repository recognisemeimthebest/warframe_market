# Claude Code 오케스트레이션 세팅 가이드

> 새 프로젝트에서 Claude Code를 체계적으로 굴리기 위한 초기 세팅 순서.
> **코드를 짤 필요 없다.** 그냥 Claude Code한테 말로 시키면 된다.

---

## 코드 없이 말로 만든다

이 가이드에 나오는 모든 것 — 훅, 스킬 매뉴얼, 에이전트, 문서 — 은 Claude Code한테 자연어로 시키면 알아서 만들어준다. 셸 스크립트를 직접 짤 필요 없다.

**예시: 이렇게 말하면 된다**

> "내가 지시를 내리면 Claude가 읽기 전에 먼저 분석하는 훅을 만들어줘. 카테고리 분류하고, 복잡한 작업이면 계획 세우라고 안내하고, 키워드 보고 관련 매뉴얼을 자동으로 로드하도록 해."

> "코드를 수정할 때마다 자동으로 위험한 부분 체크하는 훅 만들어. 에러 처리 빠졌는지, 보안 문제 없는지, 토큰 하드코딩 안 했는지 확인하게."

> "이 프로젝트에 필요한 서브에이전트들 만들어줘. 각자 자기 영역만 담당하고, 작업 끝나면 보고서 내도록 해."

> "기획서, 맥락노트, 체크리스트 3개 만들어. 세션 시작할 때 자동으로 읽도록 훅에 연결해."

핵심은 **"이런 구조가 필요해"라고 설명하면 Claude Code가 구현**한다는 것이다. 이 가이드는 "무엇을 왜 시키는지"만 알려주는 것이고, "어떻게 코딩하는지"는 몰라도 된다.

---

## 적용 전 vs 적용 후

| 상황 | 적용 전 | 적용 후 |
|------|--------|--------|
| 첫 지시 | "뭘 만드는 건가요?" 질문부터 | 기획서가 이미 컨텍스트에 있음 → 즉시 시작 |
| 새 세션 | 맥락 전부 소실 → 처음부터 설명 | 맥락노트로 30초 만에 복원 |
| 코드 품질 | 에러 처리 누락, 보안 실수 | 훅이 자동 체크 + 리마인더 |
| 다음 작업 | "뭐 하면 되죠?" | 체크리스트에서 다음 항목 자동 하이라이트 |
| 복잡한 작업 | 혼자 다 하려다 품질 저하 | 전문 에이전트에 위임 + 독립 평가 |

---

## 1단계: 기획서 작성

프로젝트 전체를 정의하는 문서를 먼저 만든다. AI가 "뭘 만드는 건지" 매번 물어보지 않게.

**포함할 것**: 목적, 기능 명세, 기술 스택, 디렉토리 구조, 데이터 소스, 마일스톤, 제약사항

**규칙**: AI는 이 파일을 읽기만 한다. 수정은 사람 또는 전담 에이전트만.

> 말로 시키기: "이 프로젝트 기획서 만들어줘. 뭘 만들 건지, 기술 스택, 기능 목록, 마일스톤 포함해서."

---

## 2단계: 권한 설정

자주 쓰는 도구에 skip permission을 건다. 매번 "허용하시겠습니까?" 팝업을 없앤다.

> 말로 시키기: "settings.json에서 Bash, Read, Write, Edit, Glob, Grep 전부 자동 허용되게 설정해줘."

---

## 3단계: UserPromptSubmit 훅 — 프롬프트 사전 분석기

사용자가 지시를 입력하면 **Claude가 읽기 전에** 훅이 먼저 분석한다.

**하는 일**:
- **세션 브리핑**: 첫 지시 시 기획서·맥락노트·체크리스트를 자동으로 컨텍스트에 주입
- **카테고리 분류**: 코드생성, 수정, 디버그, 테스트, Git 등
- **복잡도 판단**: 단어 수 + 관련 모듈 수 → 간단/중간/복잡
- **스킬 자동 로드**: 키워드·파일경로·코드패턴을 감지 → 관련 매뉴얼 자동 주입
- **에이전트 추천**: 감지된 도메인에 맞는 전문 에이전트 추천
- **이어하기 감지**: "이어서", "계속" 등 → 맥락 복원 안내

> 말로 시키기: "내가 지시를 내리면 Claude가 읽기 전에 자동으로 분석하는 훅 만들어줘. 카테고리 분류, 복잡도 판단, 키워드 보고 매뉴얼 자동 로드, 세션 시작 시 브리핑까지."

---

## 4단계: PostToolUse 훅 — 자동 기록 + 코드 체크 + 오류 분기

Write/Edit/Bash 실행 직후 자동으로 돌아가는 훅.

**하는 일 3가지**:

**(1) 수정 기록 자동 로깅** — 모든 Write/Edit가 change-log에 자동 기록.

**(2) 코드 체크** — 위험한 작업, 에러 처리 누락, 보안 문제, 프로젝트 규칙 위반을 자동 감지.

**(3) 오류 수 기반 자동 분기**:
| 이슈 수 | 대응 |
|---------|------|
| 0개 | 통과 |
| 1~2개 | "지금 바로 고쳐" — AI가 즉시 수정 |
| 3개+ | "전문 에이전트 불러" — 정밀 검토 |

> 말로 시키기: "코드를 쓰거나 수정할 때마다 자동으로 체크하는 훅 만들어. 수정 기록도 남기고, 오류가 적으면 바로 고치라고 하고, 많으면 전문 에이전트 부르라고 해."

---

## 5단계: 스킬 매뉴얼 — INDEX + 챕터 분리

프로젝트에서 지켜야 할 기술 규칙을 매뉴얼로 만들되, **전부 로드하지 않고 필요한 것만** 로드한다. 목차(INDEX) 파일에 챕터 목록과 트리거 키워드를 정리하고, 도메인별 상세 가이드를 별도 파일로 분리한다.

**활성화 방식 4가지**:
| 방식 | 예시 |
|------|------|
| 키워드 감지 | "마켓", "시세" → 마켓 챕터 로드 |
| 요청 패턴 | "만들어", "구현해" → 코드 품질 챕터 자동 추가 |
| 파일 경로 | `src/bot/` 감지 → 봇 챕터 로드 |
| 코드 패턴 | `import discord` → 봇 챕터 로드 |

> 말로 시키기: "이 프로젝트에 필요한 기술 매뉴얼을 도메인별로 챕터 나눠서 만들어줘. 훅에서 키워드 감지하면 해당 챕터만 자동으로 로드하도록 연결해."

---

## 6단계: 3대 문서 — 기획서 · 맥락노트 · 체크리스트

세션이 바뀌어도 맥락을 잃지 않기 위한 **프로젝트의 뇌**.

| 문서 | 역할 | 업데이트 시점 |
|------|------|-------------|
| **기획서** | 뭘 만들 건지 (읽기 전용) | 기획 변경 시만 |
| **맥락노트** | 왜 이렇게 결정했는지 | 매 작업 완료 후 |
| **체크리스트** | 뭘 끝냈고 뭐가 남았는지 | 매 작업 완료 후 |

**핵심**: 새 세션 시작 → 훅이 3대 문서를 자동 브리핑 → AI가 즉시 맥락 복원.

**체크리스트 규칙**: 한 번에 여러 개 체크하지 않는다. **1개 완료 → [x] 체크 → 다음 작업 시작**. 중간에 세션이 끊겨도 마지막 체크 지점까지는 신뢰 가능.

> 말로 시키기: "기획서, 맥락노트, 체크리스트 3개 문서 만들어. 세션 시작할 때 자동으로 전부 읽어서 브리핑하도록 훅에 연결하고, 작업 끝날 때마다 업데이트하도록 리마인더 넣어."

---

## 7단계: 서브에이전트 — 전문가 분업 + 독립 평가

### 도메인 에이전트
프로젝트 기능 영역별로 전문 에이전트를 만든다. 각 에이전트는:
- 자기 스킬 챕터 + 3대 문서를 읽고 시작
- 자기 영역만 작업, 영역 밖이면 해당 에이전트에게 위임
- 작업 후 보고서 출력 + 체크리스트·맥락노트 업데이트

### 평가 에이전트
**에이전트는 자기 작업을 관대하게 평가하는 경향이 있다.** 작성자와 검증자를 분리해야 한다.

| 유형 | 역할 |
|------|------|
| 코드 감사관 | 코드 품질, 보안, 에러처리를 제3자 시점에서 검증 |
| 기획 감사관 | 기획서 대비 구현 일치도 검증. 체크리스트 [x]를 맹신하지 않고 코드로 확인 |

**워크플로우**: 도메인 에이전트 작업 → 평가 에이전트 독립 검증 → 문제 시 다시 수정

> 말로 시키기: "기획서 보고 이 프로젝트에 필요한 서브에이전트들 만들어줘. 그리고 코드 감사관이랑 기획 감사관도 따로 만들어. 에이전트들이 자기 코드를 관대하게 평가하니까 검증은 별도 에이전트가 해야 해."

---

## 8단계: 에이전트 보고서 의무화

**"끝났습니다" 금지.** 모든 에이전트는 작업 완료 시 구조화된 보고서를 출력한다:

- **발견한 것** — 작업 중 알게 된 사실, 문제, 제약사항
- **수정한 것** — 어떤 파일을 어떻게 바꿨는지
- **판단 근거** — 왜 이 방식을 선택했는지, 어떤 대안이 있었는지
- **미해결 사항** — 남은 문제, 다음에 해야 할 것

**왜**: 보고서가 있어야 평가 에이전트가 검토 범위를 판단하고, 맥락노트에 기록할 내용이 정리되고, 다음 세션에서 추적이 가능하다.

> 말로 시키기: "모든 에이전트가 작업 끝나면 '끝났습니다'가 아니라 뭘 발견했고, 뭘 수정했고, 왜 그렇게 판단했는지 보고서로 보고하도록 해."

## 전체 흐름 요약

```
① 지시 내리기
   ↓
② 훅: 매뉴얼 자동 전달 + 에이전트 추천
   ↓
③ AI(또는 에이전트)가 작업 수행
   ↓
④ 훅: 수정 기록 자동 저장
   ↓
⑤ 훅: 오류 자동 검사
   ↓
⑥ 오류 분기 (0개→통과 / 1~2개→즉시 수정 / 3개+→전문 에이전트)
   ↓
⑦ 보고서 출력 (발견/수정/판단/미해결)
   ↓
⑧ 체크리스트 1개 체크 + 맥락노트 업데이트
   ↓
⑨ 다음 작업으로
```
